1. Kernel (커널):
   - 운영 체제의 핵심 부분으로, 하드웨어와 응용 프로그램 간의 통신을 관리하고 시스템 자원을 효율적으로 관리합니다. 커널은 프로세스 관리, 메모리 관리, 입출력 관리 및 파일 시스템 관리와 같은 기능을 수행합니다. 임베디드 시스템에서는 종종 리소스 제약이 있기 때문에 경량 커널이 사용되기도 합니다.

2. Interface (인터페이스):
   - 시스템 구성 요소 간의 상호 작용을 위한 접속 지점 또는 경계를 의미합니다. 인터페이스는 일반적으로 하드웨어와 소프트웨어 간의 상호 작용뿐만 아니라 소프트웨어 간의 상호 작용에도 적용됩니다. 임베디드 시스템에서는 하드웨어와 소프트웨어 간의 인터페이스가 특히 중요합니다.

3. API (Application Programming Interface):
   - 응용 프로그램이 운영 체제나 라이브러리와 상호 작용하기 위한 인터페이스를 제공합니다. API는 일반적으로 함수 또는 프로시저의 모음으로 제공되며, 개발자가 다른 소프트웨어 구성 요소와 상호 작용할 수 있도록 합니다.

4. System Call (시스템 호출):
   - 응용 프로그램이 운영 체제 커널에 서비스를 요청하는 인터페이스를 의미합니다. 시스템 호출은 일반적으로 운영 체제 기능에 접근하기 위해 사용됩니다. 예를 들어, 파일을 열거나 읽는 작업과 같은 시스템 리소스에 대한 접근에 사용됩니다.

5. OS와 Firmware (펌웨어):
   - 운영 체제(Operating System, OS)는 하드웨어와 소프트웨어 간의 인터페이스를 관리하고 시스템 리소스를 할당합니다. 펌웨어(Firmware)는 하드웨어 제어를 위한 소프트웨어로, 보통 임베디드 시스템의 초기 부트 및 하드웨어 초기화를 담당합니다.

6. Memory (메모리)**:
   - 임베디드 시스템에서는 주기억장치(RAM)와 보조기억장치(플래시 메모리 등)가 있습니다. 메모리는 프로그램 및 데이터 저장을 위해 사용됩니다.

7. NVM (Non-volatile Memory, 비휘발성 메모리):
   - 비휘발성 메모리는 전원이 꺼져도 데이터를 보존하는 메모리를 의미합니다. 주로 플래시 메모리 등이 이에 해당합니다.

8. POSIX (Portable Operating System Interface):
   - POSIX는 운영 체제 인터페이스의 표준을 정의하는 규격입니다. POSIX 규격은 이식성을 높이고 다양한 운영 체제 간의 호환성을 확보하기 위해 사용됩니다.

9. RTOS (Real-Time Operating System, 실시간 운영 체제):
   - 실시간 운영 체제는 실시간 응용 프로그램을 실행하는 데 특화된 운영 체제입니다. 임베디드 시스템에서는 주기적인 작업들이 정확한 시간에 실행되어야 하므로 RTOS가 필요할 수 있습니다.
